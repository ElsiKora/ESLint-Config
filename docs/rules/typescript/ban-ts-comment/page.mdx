# `@elsikora/typescript/ban-ts-comment`

<Callout type="info" emoji="🧭">
	**Disabled.** ElsiKora includes this rule as an intentional disabled override in at least one preset scenario.
</Callout>

## Rule Overview

Disallow `@ts-<directive>` comments or require descriptions after directives.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

## Rule Metadata

| Property           | Value                                                             |
| ------------------ | ----------------------------------------------------------------- |
| Type               | `problem`                                                         |
| Fixable            | ❌ No                                                             |
| Has suggestions    | ✅ Yes                                                            |
| Deprecated         | ❌ No                                                             |
| ESLint recommended | `{"recommended":true,"strict":[{"minimumDescriptionLength":10}]}` |
