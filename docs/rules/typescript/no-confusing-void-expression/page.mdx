# `@elsikora/typescript/no-confusing-void-expression`

<Callout type="error" emoji="🚨">
	**Error.** ElsiKora reports this rule as an error in at least one preset scenario.
</Callout>

## Rule Overview

Require expressions of type void to appear in statement position.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

## Rule Metadata

| Property           | Value     |
| ------------------ | --------- |
| Type               | `problem` |
| Fixable            | ✅ Yes    |
| Has suggestions    | ✅ Yes    |
| Deprecated         | ❌ No     |
| ESLint recommended | `strict`  |
