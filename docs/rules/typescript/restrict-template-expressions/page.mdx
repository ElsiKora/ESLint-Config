# `@elsikora/typescript/restrict-template-expressions`

<Callout type="error" emoji="🚨">
	**Error.** ElsiKora reports this rule as an error in at least one preset scenario.
</Callout>

## Rule Overview

Enforce template literal expressions to be of `string` type.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

## Rule Metadata

| Property           | Value                                                                                                                                                     |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Type               | `problem`                                                                                                                                                 |
| Fixable            | ❌ No                                                                                                                                                     |
| Has suggestions    | ❌ No                                                                                                                                                     |
| Deprecated         | ❌ No                                                                                                                                                     |
| ESLint recommended | `{"recommended":true,"strict":[{"allowAny":false,"allowBoolean":false,"allowNever":false,"allowNullish":false,"allowNumber":false,"allowRegExp":false}]}` |
