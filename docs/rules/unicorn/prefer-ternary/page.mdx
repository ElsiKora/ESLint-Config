# `@elsikora/unicorn/prefer-ternary`

<Callout type="info" emoji="🧭">
	**Disabled.** ElsiKora includes this rule as an intentional disabled override in at least one preset scenario.
</Callout>

## Rule Overview

Prefer ternary expressions over simple `if-else` statements.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

## Rule Metadata

| Property           | Value           |

| Fixable            | ✅ Yes          |
| Has suggestions    | ❌ No           |
| Deprecated         | ❌ No           |
| ESLint recommended | `unopinionated` |

## All Supported Options

The rule exposes the following ESLint option schema:

```json filename="options-schema.json" copy
[
	{
		"enum": ["always", "only-single-line"],
		"description": "Whether to always prefer ternary, or only for single-line expressions."
	}
]
```
