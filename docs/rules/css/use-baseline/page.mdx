# `@elsikora/css/use-baseline`

<Callout type="warning" emoji="⚠️">
	**Warning.** ElsiKora reports this rule as a warning in at least one preset scenario.
</Callout>

## Rule Overview

Enforce the use of baseline features.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

## Rule Metadata

| Property           | Value     |
| ------------------ | --------- |
| Type               | `problem` |
| Fixable            | ❌ No     |
| Has suggestions    | ❌ No     |
| Deprecated         | ❌ No     |
| ESLint recommended | `true`    |

## All Supported Options

The rule exposes the following ESLint option schema:

```json filename="options-schema.json" copy
[
	{
		"type": "object",
		"properties": {

						"enum": ["widely", "newly"]

					{
						"type": "integer",

		},
```

No upstream default options are exposed in the rule metadata.
