# `@elsikora/react/1/use-state`

<Callout type="error" emoji="🚨">
	**Error.** ElsiKora reports this rule as an error in at least one preset scenario.
</Callout>

## Rule Overview

Enforces correct usage of 'useState', including destructuring, symmetric naming of the value and setter, and wrapping expensive initializers in a lazy initializer function.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

import { useState } from "react";

export function Counter() {
	const counter = useState(0);

	return (
		<button type="button" onClick={() => counter[1](counter[0] + 1)}>
			{counter[0]}
		</button>
	);
}

```tsx filename="good-example.tsx" copy
import { useState } from "react";

export function Counter() {
	const [count, setCount] = useState(0);

	return (
		<button type="button" onClick={() => setCount(count + 1)}>
			{count}
		</button>
	);
}
```
