# `@elsikora/typescript/no-inferrable-types`

<Callout type="info" emoji="🧭">
	**Disabled.** ElsiKora includes this rule as an intentional disabled override in at least one preset scenario.
</Callout>

## Rule Overview

Disallow explicit type declarations for variables or parameters initialized to a number, string, or boolean.

This page describes the rule in general first, then shows the exact ElsiKora preset configuration separately.

```json filename="options-schema.json" copy
				"description": "Whether to ignore function parameters."
			},
			"ignoreProperties": {
				"type": "boolean",
				"description": "Whether to ignore class properties."
			}
		}
	}
]
```

Upstream default options:

```json filename="default-options.json" copy
[
	{
		"ignoreParameters": false,
		"ignoreProperties": false
	}
]
```
